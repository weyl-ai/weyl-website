## OverviewZ-Image Turbo is a 6B parameter model delivering sub-second generation at 1024×102
4. Perfect for:

- Rapid prototyping

- Real-time generation in tools

- High-volume batch processing

- Cost-sensitive workflows

## Basic Usage

```bashcurl -X POST "https://sync.render.weyl.ai/image/zimage/turbo/t2i?format=1024" \  -H "Authorization: Bearer $WEYL_API_KEY" \  -H "Content-Type: application/json" \  -d '{    "prompt": "portrait of a character, natural lighting"  }' \  -o output.webp

```

## Image-to-Image (i2i)**Not supported*

## Prompt Tips#

## StructureKeep prompts focused and direct:

```[subject], [style], [lighting]

```**Good:**

```portrait of a woman, soft lighting, natural colors

```**Too Complex:**

```highly detailed photorealistic portrait of a woman in her thirties with subtle makeup, golden hour lighting from the left, bokeh background with trees, shot on 85mm f/1.4, film grain

```Z-Image works best with simpler prompts. For complex scenes, use FLUX.#

## Negative PromptsEffective negatives for Z-Image:

```blur, distortion, artifacts, low quality

```Keep negative prompts short.

## BackendZ-Image Turbo only supports `nunchaku` backend (FP4 quantization on Blackwell). No backend override needed.
