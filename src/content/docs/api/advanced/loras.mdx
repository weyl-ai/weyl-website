LoRA (Low-Rank Adaptation) adapters enhance base models with specific capabilities without full fine-tuning.

## Available LoRAs

| LoRA ID          | Model Support | Effect                          ||------------------|---------------|---------------------------------|| `anti_blur`      | FLUX Dev/Dev2 | Reduces blur, sharper details   || `realism_boost`  | FLUX Dev/Dev2 | Enhanced photorealism           || `amateur_snapshot` | FLUX Dev    | Film/snapshot aesthetic         || `detail_enhance` | FLUX Dev/Dev2 | Fine detail enhancement         |*More LoRAs coming soon 

- check `/models/loras` endpoint for current list
*

## Basic Usage

```json{  "prompt": "portrait of a woman, natural lighting",  "lora": [    {      "id": "anti_blur",      "weight": 0.8    }  ]}

```

## Multiple LoRAsStack multiple adapters:

```json{  "prompt": "portrait in natural lighting",  "lora": [    {      "id": "anti_blur",      "weight": 0.8    },    {      "id": "realism_boost",      "weight": 0.75    }  ]}

```**Best practice:*

* Max 2-3 LoRAs per request. More can cause conflicts.

## Weight Tuning

| Weight | Effect                          | Recommendation      ||--------|---------------------------------|---------------------|| 0.3    | Subtle influence                | Testing             || 0.5    | Moderate effect                 | Balanced            || 0.7    | Strong effect                   | **Recommended*

*     

|| 0.9    | Very strong, may overpower      | Careful use         || 1.0+   | Maximum, possible artifacts     | ⚠️ Advanced only    |

## Anti-Blur LoRA**Reduces blur, enhances sharpness**

```json{  "prompt": "macro photography of flower petals",  "lora": [    {      "id": "anti_blur",      "weight": 0.8    }  ]}

- Portraits

- Product photography

- Macro shots

- Detail-heavy scenes**Characteristics:**

## Troubleshooting#

## Over-Application**Symptom:*

* Unnatural artifacts, over-sharpening**Solution:**

- Lower weight (try 0.6 instead of 0.9)

- Use fewer LoRAs

- Increase steps#

## Conflicting LoRAs**Symptom:*

* Strange artifacts when using multiple LoRAs**Solution:**

- Reduce weights

- Remove conflicting adapters

- Use at most 2-3 LoRAs#

## Not Enough Effect**Symptom:*

* LoRA not visible**Solution:**

- Increase weight (try 0.8-0.9)

- Ensure prompt matches LoRA purpose

- Check model compatibility

## Best Practices
1. **Start with recommended weights*

* - Usually 0.7-0.8
2. **Use 1-2 LoRAs*

* - More can conflict
3. **Match LoRA to prompt*

* - Use realism_boost with "photograph" prompts
4. **Test systematically*

* - Enable one LoRA at a time
5. **Lower guidance*

* - LoRAs work better with guidance 2.8-3.5
6. **Increase steps slightly*

* - +2-5 steps for better LoRA integration
