**Endpoint:*

* `POST /video/{family}/{model}/{task}`Generate video with immediate response. Bytes in body, CDN URL in headers.

## URL Structure

```https://sync.render.weyl.ai/video/{family}/{model}/{task}?format={format}&backend={backend}

```**Path Parameters:**

- `family` 

- Model family (`wan`, `ltx`)

- `model` 

- Model variant (`default`)

- `task` 

- Operation (`t2v`, `i2v`)**Query Parameters:**

- `format` 

- Output dimensions (`720p`, `480p`, `square`, etc)

- `backend` 

- Inference backend override (optional)

## Tasks#

## Image-to-Video (i2v)Animate a still image with motion.

```bashcurl -X POST "https://sync.render.weyl.ai/video/wan/default/i2v?format=720p" \  -H "Authorization: Bearer $WEYL_API_KEY" \  -H "Content-Type: application/json" \  -d '{    "prompt": "she slowly turns to face the camera, subtle smile",    "image": "https://example.com/portrait.jpg",    "duration": 3,    "cfg": 7.0  }' \  -o output.mp4

```**Required Fields:**

- `prompt` 

- Motion description

- `image` 

- Source image URL or data URI**Optional Fields:**

- `duration` 

- Video length in seconds (0.5-10, default: 2)

- `steps` 

- Inference steps (model-specific defaults)

- `cfg` 

- Guidance scale (6-8 for WAN)

- `seed` 

- Random seed for reproducibility

- `motion_strength` 

- Amount of motion (0-1, model-dependent)#

## Text-to-Video (t2v)Generate video from text prompt only.

```bashcurl -X POST "https://sync.render.weyl.ai/video/wan/default/t2v?format=480p" \  -H "Authorization: Bearer $WEYL_API_KEY" \  -H "Content-Type: application/json" \  -d '{    "prompt": "ocean waves crashing on beach, golden hour, aerial view",    "duration": 4  }' \  -o output.mp4

```

## Sampler & Scheduler

```json{  "prompt": "camera pushes in on character",  "image": "https://cdn.render.weyl.ai/a/abc123",  "sampler": "res_2m",  "scheduler": "bong_tangent",  "steps": 20}

```**Samplers:**

- `uni_pc` 

- Default for WAN, fast

- `euler` 

- General purpose

- `res_2m` 

- RES4LYF, better detail than uni_pc

- `res_3s` 

- Best quality, slower**Schedulers:**

- `simple` 

- Fast iteration

- `beta` 

- Good default

- `bong_tangent` 

- Bidirectional, best quality for WAN#

## Lightning Mode (WAN)Use distilled weights for 4+4 split workflow:

```json{  "prompt": "character waves at camera",  "image": "https://example.com/photo.jpg",  "lightning": true,  "shift": 5,  "cfg": 1.0}

```Requires:

- `lightning: true`

- `shift: 5`

- `cfg: 1.0`Reduces to ~8 total steps, much faster with minimal quality loss.

## Performance Tips
1. **Use 480p for iteration*

* - 2-3× faster than 720p
2. **Enable lightning mode*

* - 50% faster with minimal quality loss
3. **Reduce duration*

* - Shorter videos = faster generation
4. **Try res_2m sampler*

* - Better quality than uni_pc default
5. **Keep CFG ≤8*

* - Higher values cause color instability in WAN
